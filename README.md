# coreDB
Simple single file SQL DB. Just to learn and have fun


I decided to split it in two parts:
- kv-storage
- relational engine on top of kv

Because many modern databases do that. My idea that kv-storage layer could be used as separate product and on top of it build all SQL features will be needed.


Right now I mostly work at kv storage when I have free time (I little bit undeestimated amount of work here). But anyway it is supposed to be one of life-lasting projects. There is a lot of stuff to do here and a lot of stuff to learn.
